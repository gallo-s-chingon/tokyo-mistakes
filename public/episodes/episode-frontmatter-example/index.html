Episode Front Matter Complete Reference

This episode demonstrates all available frontmatter fields when creating podcast episodes in the Tokyo-Nord Hugo theme.

For complete documentation across all content types, see: QUICK-REFERENCE.md

Complete Episode Frontmatter Template

Below is a complete example showing ALL available frontmatter fields for podcast episodes with inline comments:

---
# ============================================================================
# REQUIRED FIELDS
# ============================================================================
title: "Episode Title"                       # REQUIRED - Episode title (H1 heading)
date: 2024-09-24T14:30:00-06:00               # REQUIRED - Hugo publish date (RFC3339 format)

# ============================================================================
# EPISODE DETAILS
# ============================================================================
episodeNumber: 42                             # local only - Episode number
season: 2                                     # local only - Season number (optional)
episodeType: "full"                           # local only - Options: full, trailer, bonus

# ============================================================================
# PODCAST INFORMATION
# ============================================================================
podcast: "Your Podcast Name"                  # local + global: params.podcast.name
                                              # Podcast name (overrides site config)
host: "Host Name"                             # local + global: params.podcast.hostName
                                              # Host name (overrides site config)
guests: ["Guest 1", "Guest 2"]                # local only - Episode guests (array)
                                              # Leave empty [] if no guests

# ============================================================================
# MEDIA FILES
# ============================================================================
featuredImage: /images/episode-42.jpg        # local only - Path to episode artwork
                                              # Appears as cover art in lists and single pages
audioUrl: ""                                  # local only - Direct audio file URL (.mp3, .m4a, etc.)
                                              # Optional: for direct audio playback
duration: "45:30"                             # local only - Episode duration
                                              # Formats: "HH:MM:SS" or "MM:SS"
                                              # Example: "1:30:00" or "45:30"

# ============================================================================
# EMBEDDED PLAYERS
# ============================================================================
embedPlayers:                                 # local only - Array of player configurations
  - type: "spotify"                           # Player type
    id: "episode-id-here"                     # Episode ID from Spotify URL
  - type: "youtube"                           # YouTube video player
    id: "video-id-here"                       # YouTube video ID
  # - type: "apple"                           # Apple Podcasts (requires both IDs)
  #   podcastId: "podcast-id"
  #   episodeId: "episode-id"

# Supported player types:
# - spotify: Spotify player
# - youtube: YouTube video embed
# - apple: Apple Podcasts player
# - Additional types may be added via theme customization

# ============================================================================
# CONTENT & SEO
# ============================================================================
tags: ["topic1", "topic2", "topic3"]          # local only - Episode topics/tags (array)
                                              # Used for categorization and discovery
summary: "Brief episode description"          # local only - Short description for cards/previews
                                              # One-liner shown in episode lists
description: "Detailed episode notes"         # local only - Full description for SEO
                                              # Used in meta tags and episode page
keywords: ["keyword1", "keyword2"]            # local only - SEO meta keywords (array)
                                              # Optional: for search optimization

# ============================================================================
# IMAGE DISPLAY OPTIONS
# ============================================================================

## Image Transparency (Makes image itself transparent)
transparency: true                            # local + global: params.images.transparency
                                              # true = make featured image transparent
                                              # false = image fully opaque (default)
transparencyAmount: 0.7                       # local + global: params.images.opacity
                                              # Range: 0.0 (fully transparent) to 1.0 (opaque)
                                              # Only applies when transparency: true

## Metadata Overlay (Shows episode metadata ON the image)
overlayMetadata: false                        # local + global: params.podcast.overlay.enable
                                              # true = metadata appears on featured image
                                              # false = metadata appears above image (default)
overlayPosition: "lower-center"               # local + global: params.podcast.overlay.position
                                              # Options: top-left, top-center, top-right,
                                              #          mid-left, mid-center, mid-right,
                                              #          lower-left, lower-center, lower-right
                                              # Only visible on single pages when overlayMetadata: true
# Note: overlayOpacity is global only in hugo.toml (no local override)
#       Set in params.podcast.overlay.opacity (default: 0.85)

## Color Overlay Filter (Colored layer on top of image)
overlay_filter: 0.5                           # local only - works on BOTH single & list pages
                                              # Formats:
                                              # - Numeric: 0.5 = 50% black overlay
                                              # - RGBA: "rgba(255, 0, 0, 0.5)" = red overlay
                                              # - Gradient: "linear-gradient(...)" = gradient effect
# Examples:
# overlay_filter: 0.3                         # 30% black overlay
# overlay_filter: "rgba(100, 0, 100, 0.5)"    # Purple overlay at 50%
# overlay_filter: "linear-gradient(to top, rgba(0,0,0,0.8), rgba(0,0,0,0))"  # Gradient

# ============================================================================
# TABLE OF CONTENTS
# ============================================================================
toc:
  enable: true                                # local + global: params.page.toc.enable
                                              # Enable table of contents
  auto: true                                  # local + global: params.page.toc.auto
                                              # Auto-collapse TOC
  keepStatic: false                           # local + global: params.page.toc.keepStatic
                                              # Keep TOC static (no auto-collapse)
# OR simple boolean format:
# toc: true                                   # Uses global params.page.toc defaults
# toc: false                                  # Disables TOC entirely

# ============================================================================
# CODE BLOCKS
# ============================================================================
code:
  copy: true                                  # local + global: params.page.code.copy
                                              # Show copy button on code blocks
  maxShownLines: 50                           # local + global: params.page.code.maxShownLines
                                              # Max lines before collapse (expands on click)

# Note: Code rendering (GoAT, Mermaid) is global only in hugo.toml
#       params.page.code.render.goat and params.page.code.render.mermaid

# ============================================================================
# PAGE FEATURES
# ============================================================================
lightgallery: false                           # local + global: params.page.lightgallery
                                              # Enable lightbox gallery for images
                                              # Recommended: false for episodes (audio focus)
linkToMarkdown: true                          # local + global: params.page.linkToMarkdown
                                              # Show link to raw markdown file
rssFullText: false                            # local + global: params.page.rssFullText
                                              # Include full text in RSS feed (vs summary)
topicsOn: true                                # local + global: params.topicsDiscussed
                                              # Show "Topics Discussed" section with tags
twemoji: false                                # local + global: params.page.twemoji
                                              # Enable Twitter emoji rendering
ruby: true                                    # local + global: params.page.ruby
                                              # Enable ruby annotations (East Asian)
fraction: true                                # local + global: params.page.fraction
                                              # Enable fraction syntax rendering
fontawesome: true                             # local + global: params.page.fontawesome
                                              # Enable FontAwesome icons

# ============================================================================
# VISIBILITY OPTIONS
# ============================================================================
draft: false                                  # local only - Draft status (true = not published)
hiddenFromHomePage: false                     # local + global: params.page.hiddenFromHomePage
                                              # Hide from home page listings
hiddenFromSearch: false                       # local + global: params.page.hiddenFromSearch
                                              # Hide from search results

# ============================================================================
# SOCIAL SHARE BUTTONS
# ============================================================================
share:
  enable: true                                # local + global: params.page.share.enable
                                              # Enable social share buttons
  # Individual platform toggles inherit from global params.page.share.*
  # Available platforms: X, Threads, Facebook, LinkedIn, WhatsApp, Viber,
  # Pinterest, Tumblr, HackerNews, Reddit, VK, Buffer, Xing, Line,
  # Instapaper, Pocket, Flipboard, Weibo, Blogger, Baidu, Odnoklassniki,
  # Evernote, Skype, Trello, Diaspora, Mix, Telegram

# ============================================================================
# COMMENTS
# ============================================================================
comment:
  enable: true                                # local + global: params.page.comment.enable
                                              # Enable comments section
  # Comment systems configured globally in hugo.toml:
  # - utterances (GitHub-based)
  # - giscus (GitHub Discussions)
  # - waline (Self-hosted)

# ============================================================================
# SEO CONFIGURATION
# ============================================================================
seo:
  images: []                                  # local + global: params.page.seo.images
                                              # Array of images for SEO/social sharing
                                              # Example: ["/images/episode.jpg", "/images/banner.jpg"]
  # Publisher info is global only:
  # - params.page.seo.publisher.name
  # - params.page.seo.publisher.logoUrl

# ============================================================================
# CUSTOM LIBRARIES (Advanced)
# ============================================================================
library:
  css:
    # customCSS = "custom.css"                # local + global: params.page.library.css
                                              # Load custom CSS libraries
  js:
    # customJS = "custom.js"                  # local + global: params.page.library.js
                                              # Load custom JS libraries

# ============================================================================
# ADVANCED HUGO FIELDS (Optional)
# ============================================================================
subtitle: ""                                  # local only - Optional subtitle
lastmod: 2024-09-24T14:30:00-06:00            # local only - Last modified date (RFC3339)
                                              # Used for SEO and sitemap
publishDate: 2024-09-24T14:30:00-06:00        # local only - Scheduled publish date (future episodes)
expiryDate: 2099-12-31T00:00:00-07:00         # local only - Content expiry date (auto-unpublish)
weight: 0                                     # local only - Content ordering weight (lower = first)
                                              # Used for manual sorting in lists
type: ""                                      # local only - Content type override (advanced)
layout: ""                                    # local only - Layout template override (advanced)
slug: "custom-url-slug"                       # local only - URL slug override
                                              # Default: auto-generated from title
url: ""                                       # local only - Full URL path override (advanced)
aliases: ["/old-url/", "/another-old/"]       # local only - URL redirects (array)
                                              # Creates redirects from old URLs
outputs: ["HTML"]                             # local only - Output formats (array)
                                              # Default: HTML, can add JSON, RSS, etc.
cascade: {}                                   # local only - Inherit settings to descendants (advanced)
menu: {}                                      # local only - Menu placement config (advanced)
isCJKLanguage: false                          # local only - CJK language processing (Chinese/Japanese/Korean)
markup: "goldmark"                            # local only - Markup renderer override (advanced)
---

Field Categories

Required vs Optional Fields

REQUIRED FIELDS:

  • title - Episode title (must be present)
  • date - Publish date in RFC3339 format (e.g., 2024-09-24T14:30:00-06:00)

HIGHLY RECOMMENDED FOR EPISODES:

  • episodeNumber - Episode number for organization
  • podcast - Podcast name (if different from site default)
  • host - Host name (if different from site default)
  • featuredImage - Episode artwork/cover art
  • duration - Episode length
  • embedPlayers - At least one player for listening
  • tags - Topics discussed in the episode
  • summary - Brief description for episode lists

OPTIONAL:

  • season - Season number (if your podcast has seasons)
  • episodeType - Usually “full”, can be “trailer” or “bonus”
  • guests - Guest names (if applicable)
  • audioUrl - Direct audio file (if not using embedded players)
  • All other fields have sensible defaults

Common Episode Configurations

Minimal Episode (Required Fields Only)

---
title: "Episode 1: Getting Started"
date: 2024-09-24T14:30:00-06:00
episodeNumber: 1
podcast: "My Podcast"
host: "Your Name"
featuredImage: /images/episode-1.jpg
duration: "45:30"
embedPlayers:
  - type: "spotify"
    id: "spotify-episode-id"
tags: ["introduction", "getting started"]
summary: "Welcome to the first episode of the podcast"
---

Understanding Embedded Players

Spotify Player

embedPlayers:
  - type: "spotify"
    id: "6F4kzTwzIqah5u75FMXc1u"  # Episode ID from Spotify URL
  • Get ID from Spotify episode URL: https://open.spotify.com/episode/6F4kzTwzIqah5u75FMXc1u
  • Shows full Spotify player with playback controls

YouTube Player

embedPlayers:
  - type: "youtube"
    id: "cIla9axQRyM"  # Video ID from YouTube URL
  • Get ID from YouTube URL: https://www.youtube.com/watch?v=cIla9axQRyM
  • Embeds full YouTube video player

Apple Podcasts Player

embedPlayers:
  - type: "apple"
    podcastId: "1234567890"
    episodeId: "1000567890123"
  • Requires both podcast ID and episode ID
  • Get from Apple Podcasts URL

Multiple Players

embedPlayers:
  - type: "spotify"
    id: "spotify-episode-id"
  - type: "youtube"
    id: "youtube-video-id"
  - type: "apple"
    podcastId: "apple-podcast-id"
    episodeId: "apple-episode-id"
  • You can embed multiple players
  • They will appear stacked on the episode page
  • Listeners can choose their preferred platform

Episode Artwork Best Practices

Featured Image

  • Recommended size: 3000x3000px (podcast standard)
  • Minimum size: 1400x1400px
  • Aspect ratio: 1:1 (square)
  • Format: JPG or PNG
  • File size: Under 500KB (optimized)

Image Guidelines

  • Use high-quality, relevant imagery
  • Ensure text is readable at small sizes
  • Include branding elements consistently
  • Test appearance on both light and dark backgrounds
  • Consider how overlays affect readability

Duration Format

Supported Formats

duration: "1:30:00"    # Hours:Minutes:Seconds
duration: "45:30"      # Minutes:Seconds
duration: "5:00"       # 5 minutes
duration: "2:15:30"    # 2 hours, 15 minutes, 30 seconds

Best Practices

  • Always include seconds (even if :00)
  • Use leading zeros for consistency: “05:30” vs “5:30”
  • Be accurate - helps with podcast apps and SEO
  • Update if you edit the episode length

Topics Discussed Feature

When topicsOn: true, the episode page will display a “Topics Discussed” section showing all tags.

Example

tags: ["productivity", "time management", "GTD", "apps", "workflow"]
topicsOn: true

This creates a visual tag cloud at the bottom of the episode page, helping:

  • Readers find related content
  • Search engines understand episode topics
  • Listeners browse by topic
  • Improve content discovery

SEO Best Practices for Episodes

Meta Description

description: "Join us as we discuss productivity systems with David Allen, creator of Getting Things Done. Learn practical tips for managing your time and tasks effectively."
  • Keep under 160 characters
  • Include episode topic and guest (if applicable)
  • Make it compelling
  • Include target keywords naturally

Keywords

keywords: ["productivity podcast", "time management", "GTD", "David Allen", "workflow"]
  • 3-7 keywords recommended
  • Mix broad and specific terms
  • Include guest names (if notable)
  • Use podcast-related keywords

Summary

summary: "David Allen shares the fundamentals of Getting Things Done"
  • One clear sentence
  • Used in episode cards/previews
  • Focus on the main value proposition

Configuration Scopes

Local Only

These fields can ONLY be set in episode frontmatter:

  • title, subtitle, date, draft
  • episodeNumber, season, episodeType
  • guests, audioUrl, duration, embedPlayers
  • description, keywords, summary, tags
  • featuredImage
  • overlay_filter

Local + Global

These fields can be set in frontmatter OR hugo.toml (local overrides global):

  • podcast → global: params.podcast.name
  • host → global: params.podcast.hostName
  • transparency → global: params.images.transparency
  • transparencyAmount → global: params.images.opacity
  • overlayMetadata → global: params.podcast.overlay.enable
  • overlayPosition → global: params.podcast.overlay.position
  • toc.* → global: params.page.toc.*
  • code.* → global: params.page.code.*
  • All feature toggles

Global Only

These fields can ONLY be set in hugo.toml:

  • Overlay opacity → params.podcast.overlay.opacity
  • Code rendering → params.page.code.render.*
  • Comment system configs → params.page.comment.*

Testing Your Episode

After creating an episode:

  1. Run Hugo development server:

    hugo server -D

  2. Check these pages:

    • Episode list: http://localhost:1313/episodes/
    • Single episode: http://localhost:1313/episodes/your-episode/
    • Tags: http://localhost:1313/tags/your-tag/

  3. Verify:

    • Episode title and metadata display correctly
    • Featured image loads and overlays work
    • Embedded players appear and function
    • Duration shows correctly
    • TOC appears if enabled
    • Topics section displays tags
    • Share buttons work

  4. Test players:

    • Spotify player loads and plays
    • YouTube player embeds correctly
    • All platforms are accessible
