3. Responsive CSS (assets/scss/responsive.scss)

.episode-header {
  // Mobile first
  padding: 1rem;

  // Tablet
  @media (min-width: 768px) {
    padding: 2rem;
  }

  // Desktop
  @media (min-width: 1024px) {
    padding: 3rem;
  }
}

4. Theme Toggle JS (assets/js/theme.js)

const toggleTheme = () => {
  const current = document.documentElement.dataset.theme;
  const next = current === 'dark' ? 'light' : 'dark';
  document.documentElement.dataset.theme = next;
  localStorage.setItem('theme', next);
};

Technical Challenges & Solutions

Challenge 1: Overlay Position Calculation

Problem: Metadata overlay wasn’t responsive across different image aspect ratios.

Solution: Used CSS Grid with named areas and percentage-based positioning. Works consistently regardless of image dimensions.

Code:

.featured-image-wrapper {
  display: grid;
  grid-template-areas:
    "top-left top-center top-right"
    "mid-left mid-center mid-right"
    "lower-left lower-center lower-right";

  .overlay-metadata {
    &.lower-left { grid-area: lower-left; }
    &.mid-center { grid-area: mid-center; }
    // ... etc
  }
}

Challenge 2: Build Performance with Large Sites

Problem: Theme’s Go templates were slow for sites with 1000+ episodes.

Solution: Optimized partial rendering with .Scratch for memoization and limited range operations.

Before: 45 seconds for 1000 pages After: 8 seconds for 1000 pages

Challenge 3: Dark Mode Flash

Problem: Brief flash of light theme before dark mode loads.

Solution: Inline critical CSS in <head> to apply theme before page render.

<script>
  (function() {
    const theme = localStorage.getItem('theme') || 'light';
    document.documentElement.dataset.theme = theme;
  })();
</script>

What I Learned

About Live Coding:

  1. Prepare but stay flexible - Have a plan but embrace tangents
  2. Explain your thinking - Narrate decisions, not just actions
  3. Embrace mistakes - Debugging live is educational
  4. Engage with chat - Viewers appreciate interaction
  5. Test everything - Murphy’s law applies to live demos

About Teaching:

  1. Analogies matter - Compare complex concepts to familiar things
  2. Show, don’t tell - Live coding beats slides
  3. Repeat key points - People tune in/out during long streams
  4. Provide resources - Give people next steps
  5. Stay enthusiastic - Energy is contagious

About Hugo Community:

  1. Hungry for advanced content - Lots of intermediate devs seeking deeper knowledge
  2. Accessibility-minded - Community values inclusive design
  3. Performance-focused - Speed and optimization are priorities
  4. Collaborative spirit - People share improvements and forks

  • Stand or use a standing desk (better energy)
  • Take a short break if stream runs long
